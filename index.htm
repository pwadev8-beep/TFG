Configuración de Dispositivos

Creación de VLANs para cada departamento en los switches:

Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Direccion_General
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name Recursos_Humanos
Switch(config-vlan)# exit
Switch(config)# vlan 30
Switch(config-vlan)# name Finanzas
Switch(config-vlan)# exit
// ... repetir para cada VLAN

Configuración de puertos troncales entre switches:

Switch(config)# interface gigabitethernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport trunk allowed vlan 10,20,30,40,50,60,100,200,250,300
Switch(config-if)# no shutdown
Switch(config-if)# exit

Habilitar el enrutamiento entre VLANs en el switch multicapa:

Switch(config)# ip routing

Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit

Switch(config)# interface vlan 20
Switch(config-if)# ip address 192.168.20.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit

// ... repetir para cada VLAN

Configuración de Telefonía IP

Configuración de Call Manager Express

Configuración básica del router para servir como Call Manager Express:

Router# configure terminal
Router(config)# telephony-service
Router(config-telephony)# max-ephones 50
Router(config-telephony)# max-dn 100
Router(config-telephony)# ip source-address 192.168.250.1
Router(config-telephony)# auto assign 1 to 100
Router(config-telephony)# exit

Router(config)# ephone-dn 1
Router(config-ephone-dn)# number 1001
Router(config-ephone-dn)# exit

Router(config)# ephone 1
Router(config-ephone)# mac-address 001B.D50A.1234
Router(config-ephone)# type 7960
Router(config-ephone)# button 1:1
Router(config-ephone)# exit

Guía de Configuración

  1. Acceder al router mediante consola o SSH
  2. Entrar en modo de configuración global con configure terminal
  3. Configurar el servicio de telefonía con telephony-service
  4. Establecer los parámetros básicos (número máximo de teléfonos, extensiones)
  5. Definir la dirección IP del servidor de telefonía
  6. Crear las extensiones (ephone-dn) con sus números
  7. Registrar los teléfonos físicos por su dirección MAC
  8. Asignar extensiones a los botones de los teléfonos
  9. Configurar las rutas de marcación para llamadas externas
Nota: Los teléfonos IP deben estar conectados a puertos PoE y en la VLAN correcta (VLAN 250).

Configuración de Cámaras de Vigilancia

Configuración de Cámaras IP

Configuración básica de una cámara IP en Cisco Packet Tracer:

  1. Arrastrar una cámara IP (Generic IP Camera) al área de trabajo
  2. Conectarla a un switch de acceso con PoE
  3. Configurar la dirección IP estática o DHCP
  4. Asegurarse de que está en la VLAN de vigilancia (300)
  5. Configurar el servidor de video para grabación
// Desde la interfaz gráfica de la cámara:
1. Hacer clic en la cámara > Config
2. Establecer dirección IP: 192.168.300.11
3. Máscara: 255.255.255.0
4. Gateway: 192.168.300.1
5. Servidor DNS: 192.168.100.11
6. Servidor de video: 192.168.300.10
7. Guardar configuración

Configuración del Servidor de Video

Configuración del servidor para gestionar las cámaras:

  1. Agregar un servidor genérico al área de trabajo
  2. Conectarlo al switch de acceso correspondiente
  3. Configurar la dirección IP estática (192.168.300.10)
  4. Instalar el software de vigilancia (simulado en Packet Tracer)
  5. Agregar las cámaras por sus direcciones IP
  6. Configurar las opciones de grabación y almacenamiento
// Desde la interfaz del servidor:
1. Hacer clic en el servidor > Services > Video Surveillance
2. Habilitar el servicio
3. Agregar nueva cámara: 192.168.300.11
4. Establecer nombre y ubicación
5. Configurar modo de grabación: continua o por detección de movimiento
6. Establecer calidad de video y retención
7. Guardar configuración

Implementación Paso a Paso

Paso 1: Diseño de la Topología

Colocar los dispositivos en el área de trabajo de Cisco Packet Tracer según la arquitectura en árbol:

  • Ubicar el switch de núcleo en el centro
  • Conectar switches de distribución al núcleo
  • Conectar switches de acceso a los de distribución
  • Colocar routers para conectividad externa
  • Ubicar los dispositivos finales en sus departamentos
Paso 2: Configuración de Dispositivos de Red

Configurar switches, routers y puntos de acceso:

  • Establecer nombres de dispositivos
  • Configurar contraseñas de acceso
  • Crear y asignar VLANs
  • Configurar enlaces troncales
  • Habilitar enrutamiento inter-VLAN
  • Configurar DHCP para asignación automática de IPs
Paso 3: Implementación de Servicios

Configurar servicios de red esenciales:

  • Servidor DHCP para asignación automática de direcciones IP
  • Servidor DNS para resolución de nombres
  • Servidor de telefonía IP (Call Manager Express)
  • Servidor de video vigilancia
  • Configurar acceso a Internet mediante NAT
Paso 4: Configuración de Dispositivos Finales

Configurar PCs, teléfonos IP y cámaras:

  • Asignar direcciones IP estáticas o verificar obtención vía DHCP
  • Configurar puertas de enlace y servidores DNS
  • Registrar teléfonos IP en el Call Manager
  • Configurar cámaras IP y asociarlas al servidor de video
  • Verificar conectividad entre dispositivos
Paso 5: Pruebas y Verificación

Realizar pruebas exhaustivas de la red:

  • Pruebas de conectividad dentro de la misma VLAN (ping)
  • Pruebas de conectividad entre diferentes VLANs
  • Verificación de funcionamiento de telefonía IP
  • Comprobación del sistema de videovigilancia
  • Pruebas de conectividad a Internet
  • Verificación de acceso inalámbrico para empleados e invitados
