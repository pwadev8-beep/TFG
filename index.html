Introducción

Esta guía proporciona instrucciones detalladas para implementar una red empresarial completa utilizando Cisco Packet Tracer. Está diseñada específicamente para apoyar el desarrollo de Trabajos de Fin de Grado (TFG) en el ámbito de redes de computadoras.

La red implementada incluirá:

  • Arquitectura jerárquica en árbol
  • Conectividad por cable e inalámbrica
  • Servicios de telefonía IP (VoIP)
  • Sistema de videovigilancia IP
  • Servicios de red esenciales (DHCP, DNS, etc.)
Recomendación para TFG: Documenta cada paso del proceso con capturas de pantalla y explica las decisiones de diseño tomadas.

Arquitectura en Árbol

La arquitectura en árbol (hierarchical network design) es un modelo escalable que divide la red en tres capas bien definidas:

Capa Núcleo

Backbone de alta velocidad para transporte entre dispositivos de distribución

Capa de Distribución

Agrega tráfico de acceso y aplica políticas (ACLs, QoS)

Capa de Acceso

Conecta usuarios finales y dispositivos periféricos

Configuración de Telefonía IP (VoIP)

La telefonía IP permite transmitir comunicaciones de voz a través de la red de datos. Sigue estos pasos para configurarla:

Paso 1: Agregar el servidor Call Manager Express

En Packet Tracer, agrega un servidor "Call Manager Express" desde la sección de dispositivos de voz.

Paso 2: Configurar el Router para VoIP

Accede a la CLI del router y configura los siguientes parámetros:

Router> enable

Router# configure terminal

Router(config)# telephony-service

Router(config-telephony)# max-dn 10

Router(config-telephony)# max-ephones 5

Router(config-telephony)# ip source-address 192.168.10.1

Router(config-telephony)# auto assign 1 to 5

Router(config-telephony)# exit

Paso 3: Configurar Números DN (Directory Number)

Router(config)# ephone-dn 1

Router(config-ephone-dn)# number 1001

Router(config-ephone-dn)# exit

Router(config)# ephone-dn 2

Router(config-ephone-dn)# number 1002

Router(config-ephone-dn)# exit

Paso 4: Registrar Teléfonos IP

Conecta teléfonos IP a los switches y configúralos para que se registren en el Call Manager Express mediante DHCP o configuración manual.

Configuración de Cámaras de Vigilancia IP

Las cámaras de vigilancia IP permiten monitorizar instalaciones a través de la red. Sigue estos pasos para configurarlas:

Paso 1: Agregar Cámaras IP

Desde la sección de dispositivos "End Devices", selecciona "IP Camera" y colócalas en el área de trabajo.

Paso 2: Configurar Direcciones IP

Asigna direcciones IP estáticas o configura DHCP para las cámaras. Es recomendable usar IPs fijas para facilitar el acceso.

Paso 3: Configurar el Servidor de Vigilancia

Agrega un servidor y configura el software de vigilancia (por ejemplo, "Surveillance System" en Packet Tracer).

1. Abre la interfaz gráfica del servidor

2. Ve a la pestaña "Services" > "Surveillance"

3. Agrega cada cámara usando su dirección IP

4. Configura las opciones de grabación y alertas

Paso 4: Verificar la Conectividad

Desde los equipos de monitorización, accede a las cámaras mediante un navegador web o el software cliente.

Sugerencia para TFG: Analiza el ancho de banda consumido por las cámaras y su impacto en la red.

Conectividad por Cable

La conectividad por cable proporciona estabilidad y alto rendimiento. Configura los switches con VLANs para segmentar la red.

Configuración de VLANs

Switch> enable

Switch# configure terminal

Switch(config)# vlan 10

Switch(config-vlan)# name Administracion

Switch(config-vlan)# exit

Switch(config)# vlan 20

Switch(config-vlan)# name Ventas

Switch(config-vlan)# exit

Switch(config)# interface fastethernet 0/1

Switch(config-if)# switchport mode access

Switch(config-if)# switchport access vlan 10

Configuración de Enlaces Troncales (Trunk)

Switch(config)# interface gigabitethernet 0/1

Switch(config-if)# switchport mode trunk

Switch(config-if)# switchport trunk allowed vlan 10,20

Conectividad Inalámbrica

Para implementar WiFi en la red empresarial, sigue estos pasos:

Paso 1: Configurar el Wireless LAN Controller (WLC)

WLC> enable

WLC# configure terminal

WLC(config)# interface management 0

WLC(config-if)# ip address 192.168.1.10 255.255.255.0

WLC(config-if)# exit

Paso 2: Crear una WLAN

WLC(config)# wlan EmpresaWIFI 1 EmpresaWIFI

WLC(config-wlan)# security wpa aks ccipher aes

WLC(config-wlan)# security wpa psk set-key ascii0 MySecurePassword

WLC(config-wlan)# no shutdown

Paso 3: Configurar los Access Points

Conecta los APs al WLC mediante el protocolo CAPWAP. Los APs deben obtener dirección IP via DHCP.

Pruebas y Verificación

Una vez configurada toda la red, realiza las siguientes pruebas para verificar el funcionamiento:

Pruebas de Conectividad
  • Ping entre dispositivos en la misma VLAN
  • Ping entre dispositivos en diferentes VLANs
  • Ping desde red cableada a inalámbrica
Pruebas de Servicios
  • Llamadas entre teléfonos IP
  • Acceso a videostream de cámaras
  • Navegación web desde clientes WiFi
Comandos de Verificación

# show vlan brief

# show interfaces trunk

# show ip route

# show running-config
